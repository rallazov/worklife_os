# worklife_os (Obsidian Vault)

This repo is an Obsidian vault (just markdown files + folders).
You can edit with any text editor and open later in Obsidian.

## Conventions
- ADRs live in: 02-Engineering/Decisions/
- Specs live in: 01-Product/ and 02-Engineering/
- Runbooks live in: 03-Ops/Runbooks/
- Agent outputs get summarized in: 04-Execution/Agent-Sessions/

## Rule of thumb
If it changes the product/system long-term → write an ADR.
If it defines behavior/UX/API → write a SPEC.
If it’s operational recovery → write a RUNBOOK.
